Atividade Avaliativa 1 – HTML & CSS

Com base nos PDFs na pasta Public — com foco em Por dentro do HTML e CSS — resolvam as questões.

Importante: além de valer nota(2), ela será meu guia para as próximas avaliações. Quanto mais natural for a sua resposta, melhor conseguirei ajustar o ritmo da turma, identificar dúvidas e aprofundar os pontos certos.

Questões

1) Verdadeiro ou falso — O HTML serve apenas para formatar visualmente o texto, sem relação com significado.

2) Complete a frase — O HTML organiza o conteúdo de forma __________, enquanto o CSS define a __________.

3) Explique — Qual a diferença entre <strong> e <b> em termos de semântica?

4) Identifique o problema

<html>
  <head>
    <title>Meu site</title>
  </head>
  <body>
    <h1>Título</h2>
  </body>
</html>

5) Explique — Por que a semântica do HTML melhora acessibilidade e SEO? Cite um exemplo de tag semântica e seu benefício.

6) Verdadeiro ou falso — O CSS é uma linguagem imperativa, pois define passo a passo a execução.

7) Complete — No CSS, a unidade px representa __________.

8) Explique — O que é o DOM e por que dizemos que o HTML é uma árvore lógica?

9) Identifique o problema (CSS)

h1 {
  color: red
  font-size: 20px;
}

10) Interpretação de código (HTML)

<ul>
  <li>Maçã</li>
  <li>Banana</li>
  <li>Laranja</li>
</ul>

11) Explique — Por que usar <header>, <nav> e <footer> ajuda na acessibilidade?

12) Identifique o problema (link)

<a>https://meusite.com</a>

13) Verdadeiro ou falso — Um mesmo id pode ser aplicado a vários elementos na mesma página.

14) Interpretação de código (link)

<a href="contato.html" target="_blank">Fale conosco</a>

15) Complete — O modelo de caixa (Box Model) é composto por: __________, __________, __________ e __________.

16) Verdadeiro ou falso — margin define o espaçamento interno de um elemento.

17) Identifique o problema (estrutura)

<head>
  <meta charset="UTF-8">
  <body>
    <h1>Título</h1>
  </body>
</head>

18) Explique — O que significa “cascata” no CSS? Cite ao menos dois critérios usados pelo navegador.

19) Interpretação de seletor

#menu {
  background-color: black;
  color: white;
}

20) Verdadeiro ou falso — O seletor universal * pode afetar todos os elementos da página.

21) Identifique o problema (CSS)

.destaque {
  background-color yellow;
}

22) Explique — Por que podemos dizer que a Web funciona como um grafo?

23) Interpretação de código (lista de definições)

<dl>
  <dt>HTML</dt>
  <dd>Linguagem de marcação para estruturar conteúdo</dd>
  <dt>CSS</dt>
  <dd>Linguagem para estilizar elementos HTML</dd>
</dl>

24) Verdadeiro ou falso — <u> dá ênfase semântica, como <em>.

25) Explique — Cite duas diferenças entre classes e IDs no CSS.

26) Interpretação/Conflito de Estilos — Qual regra “vence” e por quê?

p { color: green; }
.highlight { color: blue; }
#intro { color: red; }
<p id="intro" class="highlight">Texto de exemplo</p>

27) Explique — Como aplicar decomposição, padrões e abstração na construção de um site simples (ex.: blog)?

28) Interpretação — O que torna o hipertexto (e o elemento <a>) o “DNA” da Web? Dê um exemplo de uso correto de <a>.

29) Explique — Dê dois exemplos de decisões em HTML/CSS que impactam performance e manutenibilidade no front-end.

30) Síntese (nível avançado) — Dado o trecho a seguir, descreva a estrutura de dados (árvore DOM), a função semântica das tags e como a cascata aplicaria estilos.

<main>
  <article class="post destaque">
    <header>
      <h1 id="titulo">Engenharia de Front-end</h1>
      <p class="sub">Por que HTML é estrutura lógica?</p>
    </header>
    <section>
      <p>O <strong>DOM</strong> organiza nós como uma árvore.</p>
      <p>Links <a href="artigos.html">conectam</a> documentos.</p>
    </section>
    <footer>Atualizado em 2025</footer>
  </article>
</main>
